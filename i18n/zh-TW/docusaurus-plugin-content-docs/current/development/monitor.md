# 會話監控與強插

本章介紹微語（Bytedesk）Desktop 客服端的「會話監控 / 強插 / 接管」功能。

簡單說：如果你有管理權限，就能查看某個團隊正在進行中的對話；必要時可以介入，協助更快解決問題。

## 適用情境

- 質檢與培訓：主管即時觀察會話，必要時給出指導。
- 升級處理：遇到投訴/重大故障，主管直接介入，避免反覆轉述。
- 值班兜底：原坐席掉線或長時間未回應時，管理員接管處理。

## 核心概念

- **工作組**：一個客服團隊（例如「售前組」「售後組」）。
- **坐席**：正在接待訪客的客服。
- **監控**：進入會話查看聊天內容與狀態（一般不會改變目前由誰接待）。
- **強插**：管理員加入會話參與溝通（例如補充說明、安撫用戶）。
- **接管**：管理員把會話交給自己或其他客服來處理。

## 誰可以使用

只有被授予「工作組管理權限」的人員，才會看到並使用該功能。

如果你在 Desktop 中看不到監控入口，通常表示目前帳號沒有相關權限，需要由管理員在後台為你配置。

## Desktop 端入口與使用流程

### 入口

Desktop 客服端左側（或篩選選單）提供 **Monitor/監控** 選單。

進入後會顯示你「有權限管理」的工作組列表。

### 監控會話（典型流程）

1. 進入 **Monitor/監控** 選單。
2. 從列表中選擇一個你管理的工作組。
3. 查看該工作組的「進行中會話列表」。
4. 點擊某個會話進入詳情：
	 - 你可以先 **只查看**（監控）聊天內容與會話狀態；
	 - 如需介入處理，再執行 **強插** 或 **接管**。

## 強插與接管（行為說明）

強插與接管的目的都是「更快、更穩地把問題解決掉」，但兩者側重點不同：

- **強插**：管理員進入會話並發言，目標是「協助當前坐席把問題解決掉」。
	- 適合：需要主管補充解釋、安撫情緒、提供權限審批等。
	- 風險：多方同時發言可能造成資訊混亂，建議先內部溝通再對外回覆。

- **接管**：管理員將會話處理權轉移到自己或其他坐席。
	- 適合：原坐席無法繼續接待（離線/忙碌/技能不匹配）或需要升級處理。
	- 建議：接管後第一句話說明身份與原因，降低訪客不適。

## 使用建議

- 優先「先監控後介入」：先看清上下文，再決定是否強插/接管。
- 強插時避免搶答：盡量與原坐席分工，減少重複或衝突。
- 接管後要給訪客預期：說明你是誰、為何接手、下一步怎麼處理。

## 常見問題

### 為什麼我看不到 Monitor 選單下的工作組？

通常是因為目前帳號沒有被授予相應的管理權限。

請聯絡組織管理員在後台為你配置「可監控/可接管」的工作組。

### 管理員一定是該工作組的坐席嗎？

不一定。

有些組織會把「主管/質檢/值班負責人」設為工作組管理員，他們不一定參與日常接待，但可在關鍵時刻介入處理。
